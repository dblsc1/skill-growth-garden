# 硬件与资源:8G 跑多 agent / Docker vs firejail / CPU 瓶颈

背景:现在这台 8G 自用服务器,以后换 32G 主力编程机。CPU 是 Ryzen 3600(6核12线程)。

## 什么吃内存,什么不吃(核心认知)

Claude/Codex 的"思考"在**云端模型**里跑,本机进程只是 node 客户端,大部分时间在**等网络回包**。

| agent 在做什么 | 本机内存 |
|---|---|
| 等模型 / 思考 | 轻,~300-500MB |
| npm build / webpack | 重,1-2GB+ |
| pytest / Playwright | 重,1GB+ 每个 |

**5 个 agent 同时等模型——没问题;5 个同时构建/测试——8G 必崩。**
 
## 8G 实际预算

```
Linux 桌面 + VSCode + 浏览器  ≈ 3-4 GB
Docker daemon                ≈ 0.3 GB
留给 agent                   ≈ 3-4 GB
```
5 个思考中的 agent ≈ 2GB,塞得下;5 个同时构建 ≈ 7.5GB,爆+swap+OOM。

## 让 4-5 个 agent 在 8G 上活下来:限制并发,不限制数量

1. **多 agent ≠ 同时跑。** "代码归属"概念,不是"必须同时运行"。串行流转的图任意时刻通常只 1 个干活。
2. **真要并行就给并发上限**(LangGraph 信号量),哪怕图里 5 路并行,只放 2 个同时干重活:
   ```python
   import asyncio
   sem = asyncio.Semaphore(2)
   async def node(state):
       async with sem:
           ...  # 这里才跑构建/测试
   ```
3. **每容器内存上限**,防一个拖垮全机:`docker run --memory=1g --memory-swap=1.5g`

# 硬边界:沙箱 + CI

> ⚠️ 现行设计：SDK 跑容器内、走 Pro 订阅（非 API key）、HTTP 同步（非状态文件）。
> 见 [README.md 顶部"现行设计修正"](README.md)，以那里为准。

## 贯穿全篇的真相

> **任何"工具层"限制(Claude Code 权限规则、CLAUDE.md),只要 AI 还能跑 bash,就能绕过**(直接 cat/echo>)。
> **真沙箱——尤其"连读都不行"——只能由操作系统/容器层给,不能靠 AI 框架给。**
> 框架层做"高频快速拦截",OS/容器层做"物理死线"。两层配合。

## 软 → 硬 全景图

| 要的保证 | 软约束(高频、可绕) | 硬边界(物理、不可绕) |
|---|---|---|
| 代码质量/不改坏 | Stop hook 跑 `make check` | **CI + 分支保护** |
| 端到端不回归 | 本地跑 Playwright | **CI 里的 Playwright** |
| 只能改自己那块 | Claude 权限 deny 规则 | **容器只挂自己的卷** |
| 别处连读都不行 | (做不到) | **容器不挂载 / OS 权限 / firejail** |
| 多 agent 分权 | 各包 CLAUDE.md | **各 agent 各沙箱 + contracts 只读** |

## 硬边界一:测试工具变成边界(CI / Playwright)

测试工具本身不是边界,装在哪个"闸"上才是关键。两层:

### 层一:本地快闸(Claude Code Hooks)— 快速反馈
```jsonc
// .claude/settings.json
{
  "hooks": {
    "Stop": [{
      "matcher": "",
      "hooks": [{ "type": "command", "command": "make check" }]
    }]
  }
}
```
hook 非零退出 → Claude 收到失败 → 自己修好再交。

### 层二:服务端死闸(CI + 分支保护)— AI 绕不过
在 GitHub 服务器上,本地 shell 碰不到。

`.github/workflows/ci.yml`:
```yaml
name: ci
on: [pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - run: make check        # lint + 类型 + 单元测试
      - run: make e2e          # Playwright 端到端
```

开分支保护(这步才让 CI 变"硬"):
```bash
gh api -X PUT repos/{owner}/{repo}/branches/main/protection \
  -f "required_status_checks[strict]=true" \
  -f "required_status_checks[contexts][]=check" \
  -f "enforce_admins=true" \
  -f "required_pull_request_reviews[required_approving_review_count]=1" \
  -f "restrictions=null"
```
效果:CI 不绿 → 合并按钮灰的(包括你和 AI);必须有人 approve;连你自己也不能绕。
**Playwright 放 CI** = 跨 TS/Python 的端到端保险。
用 `CODEOWNERS` 规定 `packages/contracts/` 改动必须本人 approve(改共享契约必须过人)。

## 硬边界二:AI 沙箱(只能改自己那块,别处连读都不行)

从软到硬四级,真要"连读都不行"直接上 Level 3。

### Level 1:Claude Code 权限规则(挡"乖乖走工具"的路)
```jsonc
// .claude/settings.json
{
  "permissions": {
    "deny": [
      "Read(./packages/frontend/**)",
      "Edit(./packages/frontend/**)",
      "Write(./packages/frontend/**)",
      "Read(./.env)",
      "Read(./secrets/**)"
    ]
  }
}
```
漏洞:允许 Bash 就能绕。只拦诚实路径,不是真沙箱。

### Level 2:git worktree(隔离写 + 爆炸半径,不隔离读)
每任务独立 worktree,改坏不污染主干。但含整个仓库,读不隔离。

### Level 3:容器沙箱(★ 真正"连读都不行")
卷只挂该看的目录,禁区根本不在容器文件系统里。**现行做法**：不传 API key，挂 Pro 登录态走订阅（见 `docker/docker-compose.agents.yml`）：
```bash
docker run --rm -it \
  -v "$PWD/backend":/workspace/backend \                 # 可读写
  -v "$PWD/contracts":/workspace/contracts:ro \          # 只读
  -v "$PWD/docs":/workspace/docs:ro \                    # 全局文档只读
  -v "$HOME/.claude/.credentials.json":/root/.claude/.credentials.json:ro \  # Pro 订阅登录态
  -w /workspace agent-sandbox python runner.py
# 注意：容器内 pop ANTHROPIC_API_KEY，强制走挂进来的订阅凭证，不走 API 账单。
```
- 没挂的目录在容器里**不存在**,任何手段读不到。
- `:ro` = 能看不能改,OS 物理保证。
- AI 在容器里有 shell 也越不出容器。
- (可选)锁网络只放行模型 API,防外泄(抄 Anthropic 官方 devcontainer 的 init-firewall.sh)。

### Level 4:独立 Unix 用户
轻量版:专用用户 + chmod/chown,对禁区无读写权限。

> 8G 机器 MVP:先 Level 1 + Level 2,够用零成本。需要真读隔离时上容器或 firejail(见 04)。

## 硬边界三:多 agent 权限拆分

**前提:后端先拆子包(边界即文件夹)。**
```
packages/backend/
├─ contracts-internal/   # 内部接口(只读共享)
├─ auth/        ← agent A
├─ grading/     ← agent B
└─ content/     ← agent C
```
边界干净时,"一分为二"= 多挂一个卷/多写一条 deny:
```
agent A:挂 auth/(读写) + contracts(只读)
agent B:挂 grading/(读写) + contracts(只读)
→ 互相看不见实现,只能隔着 contracts 协作
```

**关键洞察:agent 之间"互相失明"靠契约,不靠看代码。** 若 B 非得读 A 内部才能干活 → 边界切错了。**沙箱是架构质量的探针。**

- 边界干净:极方便,加 agent = 加子包 + 挂载配置 + 该包 CLAUDE.md,线性扩展。
- 协调成本几乎只在改 contracts → 规矩:**改 contracts 必须人审**(interrupt 或 PR 强制 review),改自己子包可放权。

## 一句话总线

> 高频拦截用框架层(hooks/权限),物理死线用 OS/容器层,协作边界用 contracts。三者各管一段,缺一层就有缝。
